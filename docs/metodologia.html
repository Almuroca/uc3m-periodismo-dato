Metodología de la página web

El objetivo de esta página web consiste en reunir todas las actividades realizadas durante el curso 2020/2021.

Requisitos necesarios:

Antes de lanzar esta página web es necesario cumplir los siguientes pasos:

  1. Pandoc: para poder realizar este ejercicio he tenido que pasar todos los archivos que tenía en formato markdown a html. Para ello, tuve que instalarme Pandoc desde la terminal mediante el comando sudo apt install, el comando que utiliza mi sistema operativo (Linux) para instalar paquetes, el resto de sistemas operativos utilizan otros comandos como ("apt cyg install" en Microsoft Windows o "brew install" en Mac OS X).
  2. Bootstrap: en la página de Bootstrap me descargué los ejemplos, simplemente tuve que seleccionar en la parte superior el apartado de ejemplos y la opción de Download examples. Una vez estaban instalados utilicé el comando unzip (previamente instalado) para extraer los que a mi me interesaban que se encontraban en la carpeta assets/dist/. En la misma, había otras dos: js y css. La primera de ellas hace referencia al lenguaje java script y la segunda a la hoja de estilo que vamos a utilizar. Asimismo extraje la carpeta sticky footer navbar donde se encontraba el index.hmtl que tomé como referencia para realizar la transformación de markdown a html.
  3. Github, carpetas: cuando ya tenía todo lo que necesitaba (Pandoc, ejemplos de Bootstrap) cree una nueva carpeta en mi repositorio uc3m-periodismo-dato. Para ello, desde la terminal me situé en mi repositorio mediante el comado (cd=change directory) y con el comando (mkdir=make directory) hice la carpeta docs/. Dentro de ella mediante el comando (cp=copy) introduje la carpeta css y js. En esta misma carpeta (docs/) también copié la carpeta de imágenes que tenía, (cp ../uc3m-periodismo-dato/imagenes docs/). Esta orden consiste en copiar la carpeta que tenía en mi repositorio a la nueva carpeta creada /docs. Esta acción me permitió hacer referencia a estas imágenes en mis archivos HTML ya que se encontraba dentro de docs/. Asimismo, incluí en la carpeta css la carpeta de sticky-footer-navbar.css que permitía jugar con la parte superior de la página web donde yo incluí cada una de mis actividades. En esta carpeta también añadí cada uno de mis ejercicios, ya en formato html (se explicará a continuación,) y añadí una nueva carpeta que recibió el nombre de src para introducir todos los csv que había creado con mi Open Refine. De esta forma, en mi carpeta docs/ tenía cuatro carpetas (css, imagenes, js, src) y los ejercicios transformados en html junto a mi index.html y mi metodologia.html
  4. Github, pages: en el apartado Settings de mi repositorio me situé en la opción Pages. Ahí hay una ocpión que te permite establecer dónde se va a construir tu página web. Por eso, yo elegí como branchla opción main y la carpeta /docs (desde donde se construirá mi página web). Una vez elegí las opciones le di a guardar y se generó un link que me permite acceder a mi página web.

Pasos a seguir:

  1. Transformar todos mis archivos .md en .html mediante el comando Pandoc. Lo único que tuve que hacer fue situarme en mi repositorio donde reunía todos mis ejercicios en formato markdown. Como ya he mencionado, lo hice mediante el comando cd, me sitúe en mi repositorio y le dije a mi terminal que transformara a html los archivos que yo le decía. La orden enviada: pandoc primera-actividad-comentario-critico.md
  2. Copiar y pegar el contenido html en mi index.html en la parte begin content. Hay dos formas de realizar este paso. La primera de ellas, consiste en dividir el archivo index.hml en tres partes: una cabecera, un cuerpo y un pie. De esta forma, modificas la cabecera y el pie, e introduces en el cuerpo el contenido html de cada una de tus prácticas. A continuación, mediante el comando cat unes todas estas partes. Solo modificas una vez la cabecera y el pie, que será lo que utilizas todo el tiempo y el cuerpo lo vas modificando en función del contenido. Sin embargo, cuando yo realicé este ejercicio no se me ocurrió esta acción por lo que recurrí a otra táctica. Yo desde mi terminal, situada en mi repositorio y en mi carpeta docs, ponía nano (mi editor de texto) index.html, y modificaba el contenido en la parte de begin content. Realmente solo tuve que modificar una vez la cabecera de index.html donde cambié el menú superior y el pie donde puse "Periodismo de Datos, UC3M". Así pues, son dos formas parecidas de llegar al mismo resultado.
  3. Modificar en el html algunas líneas.(Los cambios que he realizado aparecen indicados en mi documento hmtl, en el apartado href y script).Si no introducimos estos cambios la página web no tendrá el diseño que queremos. Por eso, tuve que indicarle a mi página web dónde se encontraba el estilo que yo quería usar, es decir, modifiqué las referencias que aparecen de forma automática por las que yo quería.
  4. Modificar ruta de imagen: el archivo que copiamos con formato markdown contiene imágenes que hacen referencia a una carpeta determinada porque se encuentran en nuestro repositorio. Sin embargo, cuando las copiamos en html tenemos que modificar la referencia para que muestre las imágenes que nos interesan, es decir, las que se encuentran en nuestra carpeta docs/ en otra carpeta que recibe el nombre de "imagenes".
  5. Guardar. Cuando realicé todos estos cambios en mi archivo index.html guardé cada actividad con un nombre diferente. De esta forma, se quedaban mi archivo index.html y los que iba creando de forma progresiva.

¿Cómo acceder a la página web?

  • Atajo de teclado: control+o, que se utiliza para abrir una URL, un documento, una imagen desde el ordenador. Para ello, debes situarte en el navegador de internet, presionar las teclas mencionadas y elegir aquello que quieres abrir, en mi caso un archivo html.
  • Comando pyhton3 -m http.server para ver el puerto y el http de mi página web. Para ello, me tuve que descargar el comando en mi terminal mediante sudo apt install, que es el comando que utiliza el sistema operativo Linux, es decir, el mío. Una vez lanzas el comando solo tienes que copiar el puerto que te indica, además resulta muy útil para ver si tienes que modificar algún aspecto de tu página web.
  • Github: te sitúas en Settings luego seleccionas Pages y te aparece el link en la parte superior, mediante el cual puedes acceder directamente.

Paso final

Cuando terminé todo este proceso, tuve que comprimir todo mi repositorio, "uc3m-periodismo-dato" en una carpeta zip. Para ello, mediante el comando zip (previamente instalado con sudo apt install) desde fuera de la carpeta puse "zip -r cursocompleto.zip docs/." Esto quiere decir que quería que mi carpeta se llamara "Curso Completo" y la carpeta que quería zipear era docs. A continuación, copie ese .zip en mi carpeta docs, lo añadí a mi repositorio git y lo subí a Github mediante las opciones ya mencionadas (git add, git commit -m, git push origin main).
